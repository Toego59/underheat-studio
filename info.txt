==========================================
LOGIN SYSTEM FIX — May 11, 2026
==========================================

ISSUE DIAGNOSED:
The login was returning "Network error: Unexpected token 'B', 'Bad Gatewa'... is not valid JSON"

ROOT CAUSE:
frontend/proxy-simple.js was trying to connect to BACKEND_PORT 3000, but the actual backend server is running on port 4000. This caused a connection error that returned plain text "Bad Gateway" instead of JSON, which the frontend JavaScript couldn't parse.

FIXES APPLIED:

1. PROXY SERVER (frontend/proxy-simple.js)
   - Changed BACKEND_PORT from 3000 to 4000
   - Changed error response from plain text to JSON format
   - Now returns: { success: false, message: "Bad Gateway: ..." }

2. BACKEND AUTH (backend/server.js)
   - Updated user storage to use "role" field instead of "isAdmin" boolean
   - Updated login to support both old and new formats for backwards compatibility
   - Added new /api/auth/setup endpoint to create founder account (one-time)
   - Register endpoint now creates users with role "user"
   - Login endpoint returns proper role in JWT token and response

ROLE SYSTEM STRUCTURE:
- "user" — Regular registered user, can submit feedback
- "admin" — Has admin panel access (reserved for future use)
- "founder" — Full system access, can see admin notes and founder-only features

CURRENT ARCHITECTURE:
- Frontend: Static HTML/JS pages served via proxy-simple.js (port 5500)
- Backend: Express.js server with Cloudflare KV storage (port 4000)
- Auth: JWT tokens with 7-day expiration
- User data: Stored in Cloudflare KV namespace
- Passwords: SHA-256 hashed (consider upgrading to bcrypt for production)

IMPORTANT NOTES:
